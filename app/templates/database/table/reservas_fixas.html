Gerenciamento da Tabela Reservas FIXAS

A tabela reservas_fixas possui os seguintes campos:

Campo Tipo Descrição Valores Possíveis
id_reserva_fixa INT (auto_increment, PK) Identificador único da reserva semestral no sistema -
id_responsavel INT (FK) Chave estrangeira para a tabela pessoas -
id_responsavel_especial INT (FK) Chave estrangeira para a tabela usuarios_especiais -
tipo_responsavel INT (campo calculado - não existe fisicamente no banco) Indica qual responsável se aplica à reserva, derivado dos campos id_responsavel e id_responsavel_especial.
  • 0 = Pessoa (id_responsavel)
  • 1 = Usuário Especial (id_responsavel_especial)
  • 2 = Ambos
  • 3 = Nenhum
id_reserva_local INT (FK) Chave estrangeira para a tabela locais -
id_reserva_aula INT (FK) Chave estrangeira para a tabela aulas_ativas -
id_reserva_semestre INT (FK) Chave estrangeira para a tabela semestres -
finalidade_reserva ENUM Define a qual fim se destina a reserva
  • 'GRADUACAO' = destinado à graduação
  • 'ESPECIALIZACAO' = destinado à especialização
  • 'EAD' = destinado à EAD
  • 'NAPTI' = destinado ao NAPTI
  • 'CURSO' = destinado a cursos
  • 'USO_DOS_ALUNOS' = destinado ao uso geral de alunos/visitantes
observacoes TEXT Observação sobre a reserva (exemplo: softwares necessários) -
descricao VARCHAR(100) Descrição da reserva (usada para minicurso ou casos específicos) -
Nota: O campo tipo_responsavel não existe fisicamente no banco de dados. Ele é calculado pela aplicação a partir dos campos id_responsavel e id_responsavel_especial, servindo apenas para indicar de forma simplificada qual responsável está associado à reserva.
Esse campo pode ser removido do modelo físico sem impacto, desde que a lógica de cálculo seja mantida no nível da aplicação.
