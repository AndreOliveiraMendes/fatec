Gerenciamento da Tabela Finalidade de Reservas

A tabela finalidade_reservas possui os seguintes campos:

Campo Tipo Descrição Valores Possíveis
id_finalidade INT (auto_increment, PK) Identificador único da finalidade da reserva -
nome VARCHAR(50) Nome da finalidade (ex: Graduação, Curso, NAPTI) Texto livre (recomendado padronizar)
ativo TINYINT(1) Indica se a finalidade está ativa no sistema
  • 1 = Ativo
  • 0 = Inativo
descricao TEXT Descrição detalhada da finalidade da reserva -
config JSON Campo para configuração dinâmica da finalidade. Pode ser utilizado para definir comportamento da interface ou regras específicas sem necessidade de alterar o código.
  • {"template": "%responsavel_auto em %local (%aula)"}
  • {"template": "%responsavel_auto - %descricao"}
  • {"template": "%local | %aula | %observacoes"}
Nota: O campo config utiliza o tipo JSON nativo do MySQL 5.7+. Ele permite armazenar configurações flexíveis para cada finalidade, evitando a necessidade de alterar a estrutura do banco ou regras fixas na aplicação.

Recomenda-se utilizar esse campo apenas para configurações simples e controladas, evitando lógica de negócio complexa diretamente no banco de dados.
